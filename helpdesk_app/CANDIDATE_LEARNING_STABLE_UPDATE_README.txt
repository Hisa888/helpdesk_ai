# 学習安定版アップデート

## 目的
「もしかしてこれ？」候補のクリック学習が強く効きすぎて、1回の誤クリックで次回以降の回答が固定化される問題を防止する。

## 修正内容
- 学習補正を最大 +0.08 に制限
- 1回クリックあたりの補正を +0.02 に弱める
- 元スコアが 0.18 未満で、かつ概念一致もない候補は学習補正しない
- セッションログとCSVログの二重カウントを防止
- 元のTF-IDF/正規化/候補抽出ロジックを優先し、学習は順位を少し後押しする用途に限定

## 確認方法
1. 「モニター死んだ」と入力
2. 候補をクリック
3. 再度「モニター死んだ」と入力
4. 不自然に 90% 超へ跳ね上がらず、正しい候補が少し上位に出ることを確認
