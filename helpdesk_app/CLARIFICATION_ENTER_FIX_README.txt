追加質問入力のEnter送信修正
==============================

修正内容:
- st.chat_input の key を通常質問/追加情報で切り替えず、常に main_chat_input に固定しました。
- 追加質問を開始した直後に一度 rerun するようにして、入力欄の placeholder と session_state を同期します。
- これにより「アプリ」→ Enter → 追加質問 →「インストール」→ Enter の流れで、
  「インストール」が表示・処理されない問題を防止します。

確認した想定動作:
1. 「アプリ」と入力
2. 追加質問が表示される
3. 入力欄に「追加情報を入力してください」と表示される
4. 「インストール」と入力して Enter
5. チャット履歴に「アプリ / 補足情報：インストール」が表示され、
   検索用には「アプリ インストール」として再検索される
