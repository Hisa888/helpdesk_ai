操作列のルール
--------------
+     : 追加。FAQ_IDは使用せず、アプリ側で新しいFAQ_IDを自動採番します。
空白  : 更新。FAQ_IDに一致する既存FAQを上書きします。
-     : 削除。FAQ_IDに一致する既存FAQを削除します。

更新日・更新者のルール
----------------------
更新日 : FAQを反映した現在日時をアプリ側で自動設定します。
更新者 : ログイン中のログインIDをアプリ側で自動設定します。
備考   : Excel/CSVの「備考」列の内容を取り込みます。

重要
----
Excel/CSVに「更新日」「更新者」が入力されていても、FAQ反映時にはアプリ側の値で上書きします。
これは、誰でもExcel上で更新者を書き換えられてしまう問題を避けるためです。

ログインIDの設定
--------------
FAQの「更新者」には、ログイン時に入力したログインIDを保存します。

単一管理者の場合:
ADMIN_LOGIN_ID=admin
ADMIN_PASSWORD=任意のパスワード

複数管理者の場合:
ADMIN_USERS=sato:password1,tanaka:password2

例:
ログインID sato でログインしてFAQを反映した場合、更新者には sato が入ります。

履歴ログ
--------
FAQ反映時、下記ファイルに履歴を追記します。

runtime_data/faq_import_history.csv

履歴列:
日時, 更新者, 操作, FAQ_ID, 質問, 結果, 備考
