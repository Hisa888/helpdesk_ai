# 安全検索エンジン強化 反映内容

今回の修正では、FAQ検索を「スコアが高いから即回答」ではなく、下記の安全判定を通すようにしました。

## 追加した制御

1. 業務用語一致チェック
   - 例: 起案書 / システム導入 / サイトアクセス許可 / アプリインストール / アカウントロック など
   - 「申請書」「書式」だけで別FAQを誤採用しないように補正します。

2. 除外条件チェック
   - FAQに「除外キーワード」を持たせた場合、その語が質問に含まれる候補は強く減点します。

3. 必須キーワードチェック
   - FAQに「必須キーワード」を持たせた場合、その語が質問に無い候補は強く減点します。

4. 曖昧な質問は候補表示
   - 例: 「パソコンがロックされました」は、アカウントロック / 画面ロック / BitLocker / MDM などに分かれるため、自動回答せず候補表示へ回します。

5. 自動回答禁止・候補表示優先
   - FAQ行に「自動回答許可」「候補表示優先」を持たせることで、行単位の安全制御ができます。

6. 1位と2位の差分チェック
   - スコア差が小さい場合は、嘘回答を避けて「もしかしてこれですか？」に回します。

## 追加対応したFAQ列

既存列に加えて、任意で下記列を利用できます。空欄でも動作します。

- 必須キーワード
- 除外キーワード
- 曖昧判定キーワード
- 候補表示優先
- 自動回答許可

## 検索精度設定に追加した項目

管理者画面の「検索精度設定」→「詳細設定」内に、安全検索ガードを追加しました。

- 安全検索ガードを有効にする
- 1位と2位の最低差分
- 高信頼スコア
- 低スコア候補表示しきい値
- もしかして候補数
- 業務用語一致ボーナス
- 業務用語不一致ペナルティ
- 必須キーワード不一致ペナルティ
- 除外キーワード一致ペナルティ

## 確認した検索例

- 起案書の書式はありますか？
  - FAQ-000025 起案書の書式・資料はどこにありますか？ が上位になります。

- システム導入するための申請書はどれですか？
  - システム導入の申請書FAQが上位になります。

- サイトアクセス許可の申請書はどれですか？
  - サイトアクセス許可の申請書FAQが上位になります。

- パソコンがロックされました
  - 自動回答せず候補表示に回ります。アカウントロック系FAQを候補に出しやすくしています。

## 注意

既存のFAQデータは消さず、既存列も維持しています。
検索精度設定を変更した後は、Streamlitを再起動またはキャッシュクリアしてください。
