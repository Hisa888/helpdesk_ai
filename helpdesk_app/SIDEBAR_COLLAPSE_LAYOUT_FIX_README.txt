# 左上「<<」折りたたみ時のレイアウト崩れ修正

## 修正対象
- helpdesk_app/modules/app_bootstrap.py

## 修正内容
1. Streamlit 標準の左メニュー折りたたみ操作（左上の「<< / >>」）を押した際、
   独自の左メニュー幅 CSS とドラッグ用リサイズバーが残り、画面左側に大きな余白が出る問題を修正。
2. サイドバーが折りたたまれた状態を JavaScript で検知し、以下を自動制御。
   - `--user-sidebar-width` を一時的に `0px` に変更
   - 独自の青いリサイズバー `#oai-sidebar-resizer` を非表示
   - メイン画面側の不要な空白を抑制
3. 「>>」でサイドバーを戻す際は、保存済みの左メニュー幅を復元。
4. `st.set_page_config` に `initial_sidebar_state="expanded"` を追加し、初期表示では左メニューを開いた状態に統一。
5. Streamlit の再描画ごとに同じイベントリスナーが重複登録されないよう、既存ハンドラをクリーンアップする処理を追加。

## 期待される動き
- 左上の「<<」を押しても、画面左側に大きな空白や青い縦バーが残らない。
- 折りたたみ後はメイン画面が自然に広がる。
- 「>>」で戻すと、以前の左メニュー幅に戻る。
- 既存のFAQ、RAG、管理画面、ログ、PDF、音声入力などの機能は変更していません。
