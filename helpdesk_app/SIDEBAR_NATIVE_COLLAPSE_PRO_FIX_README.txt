# サイドバー折りたたみ／再表示 修正内容

## 対象
左上の Streamlit 標準ボタン「<<」「>>」で、管理メニュー（サイドバー）を閉じたり開いたりする動作。

## 修正した問題
- 「<<」で閉じた後、左側に余白やリサイズバーが残る
- 「>>」が2つ表示される
- 独自ボタンと Streamlit 標準ボタンが競合し、押しても開かずに戻る

## 修正方針
独自の再表示ボタンを追加する方式はやめ、Streamlit 標準の「<<」「>>」をそのまま使う方式に戻しました。
その上で、独自実装しているサイドバー幅変更バーだけを、サイドバー折りたたみ状態では自動的に非表示にします。

## 主な修正ファイル
- helpdesk_app/modules/app_bootstrap.py

## 実装内容
- サイドバー幅指定を `aria-expanded="false"` のときには適用しないように変更
- サイドバーが閉じているときは幅を 0 にして、左余白が残らないように変更
- 独自の青いリサイズバー `#oai-sidebar-resizer` は閉じている間だけ非表示
- JavaScript の MutationObserver で Streamlit の開閉状態を監視
- 独自の「>>」ボタンは追加しないため、二重表示・競合を回避

## 確認観点
1. 起動直後、左メニューが表示される
2. 左上の「<<」を押すと、左メニューが閉じる
3. 閉じた状態で青いリサイズバーが残らない
4. 「>>」は Streamlit 標準のものだけ表示される
5. 「>>」を押すと、左メニューが再表示される
6. 再表示後、左メニュー幅のドラッグ変更が使える
