# サイドバー折りたたみ時の「>>」二重表示 修正版

## 対象
- `helpdesk_app/modules/app_bootstrap.py`

## 修正内容
左上の「<<」で管理メニューを閉じた後、Streamlit標準の「>>」と独自の「>>」が2つ表示される問題を修正しました。

今回の修正では、Streamlitのバージョン差によって `data-testid` や `aria-label` が付かない標準ボタンにも対応するため、以下の条件で標準ボタンを検出します。

- 左上付近に表示される小さなボタン
- `>>` / `≫` / `»` などの復帰アイコン
- SVGアイコンのみで描画されるStreamlit標準ボタン
- `sidebar` / `サイドバー` 系の属性を持つボタン

折りたたみ時は、Streamlit標準の復帰ボタンを非表示にし、独自の「>>」ボタンだけを表示します。

## 維持している動作
- 左メニューを閉じたとき、左余白・青い縦バーを残さない
- 独自の「>>」ボタンから管理メニューを再表示できる
- 管理メニュー表示中は、左右ドラッグで幅変更できる
- 既存機能は削除していません

## 確認コマンド
```bash
python -m compileall -q helpdesk_app
```
