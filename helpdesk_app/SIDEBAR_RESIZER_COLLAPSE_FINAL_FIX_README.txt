# サイドバー折りたたみ時の青い縦バー残り修正

## 修正対象
- helpdesk_app/modules/app_bootstrap.py

## 修正内容
Streamlit標準のサイドバー開閉ボタン（<< / >>）だけを使う方針は維持し、独自の再表示ボタンは追加しません。

今回、左メニューを閉じたあとに、幅調整用の青い縦バーだけが画面中央寄りに残る問題を修正しました。

主な対応は以下です。

- Streamlit標準の「>>」ボタンを複数条件で検出
  - data-testid="collapsedControl"
  - aria-label / title に Open / Expand / Show sidebar を含むケース
  - 表示文字が >> / ≫ / » / › のケース
- サイドバーが閉じている間は `#oai-sidebar-resizer` を完全に非表示
- サイドバーが開いている間は、青い縦バーを実際のサイドバー右端に追従
- MutationObserver と短周期同期で、クリック後の状態変化にも追従

## 確認ポイント
1. 左上の `<<` で管理メニューを閉じる
2. Streamlit標準の `>>` だけが左上に表示される
3. 青い縦バーが画面中央寄りに残らない
4. `>>` を押すと管理メニューが再表示される
5. 管理メニュー表示中だけ青い縦バーがメニュー右端に表示される
