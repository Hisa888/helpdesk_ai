# サイドバー開閉の安定化修正

## 修正方針
前回までの独自「>>」ボタン、青い縦リサイズバー、MutationObserver による監視処理を廃止し、Streamlit 標準の「<< / >>」だけでサイドバーを開閉する安定方式に戻しました。

## 修正理由
独自ボタンと Streamlit 標準ボタンを同時に制御すると、以下の不具合が起きやすいためです。

- 「>>」が2つ表示される
- 「>>」を押しても開かず、閉じた状態に戻る
- 青い縦バーが中央付近に残る
- MutationObserver と setInterval が再描画を繰り返し、画面が応答しなくなる

## 修正内容
- `helpdesk_app/modules/app_bootstrap.py` を修正
- 独自の再表示ボタンを使わない
- 独自の青い縦リサイズバーを無効化
- 旧バージョンで残った監視処理とタイマーを停止
- 旧バージョンで残ったDOM要素を削除
- Streamlit標準の「>>」を左上に寄せるCSSだけを適用
- サイドバー幅のCSSは開いている時だけ適用

## 確認ポイント
1. アプリ起動後、通常表示されること
2. 左上の「<<」で管理メニューを閉じられること
3. 左上の「>>」で管理メニューを開けること
4. 「>>」が1つだけ表示されること
5. 青い縦バーが中央に残らないこと
6. 画面が固まらないこと

## 注意
ブラウザ側に前回のJavaScript状態が残っている場合があります。適用後は一度、ブラウザで Ctrl + F5 か、Streamlit の Stop → 再起動を実施してください。
